# GARAGE F1

Одностраничный сайт автосервиса GARAGE F1 в Батуми. Проект доработан на основе экспортированного Stitch-макета `code.html`: сохранены тёмная палитра, красные акценты, крупная типографика, угловатые блоки и общая композиционная логика.

## Стек

- Vite
- Vanilla JavaScript (ES modules)
- HTML5 и CSS3
- локальные шрифты Montserrat и Inter из npm-пакетов `@fontsource`

## Запуск

Требуется Node.js 20.19+ или 22.12+.

```bash
npm install
npm run dev
```

Vite выведет локальный адрес сайта в терминале.

## Production-сборка

```bash
npm run build
npm run preview
```

Готовые статические файлы создаются в папке `dist`.

## Где редактировать данные

Все контакты, навигация, услуги, цены, социальные ссылки и история владельца находятся в одном файле:

```text
src/data.js
```

### Интервью владельца

В `src/data.js` массив `ownerInterview` содержит вопросы и выверенные ответы владельца GARAGE F1. Он выводится в разделе личной истории как доступный список вопросов и ответов. Чтобы обновить биографию, даты или формулировки, редактируйте этот массив — разметка раздела останется без изменений.

Основная разметка и интерактивная логика находятся в `src/main.js`, стили — в `src/styles.css`, SEO-метаданные и JSON-LD — в `index.html`.

### Как редактировать историю владельца

Раздел «Гоночная ДНК GARAGE F1» использует массив `gallery` в `src/data.js`. Для каждой фотографии доступны поля:

- `src` — путь к оптимизированному изображению;
- `chapter` — одна из глав истории;
- `title` — короткий заголовок текущего кадра;
- `caption` — нейтральная подпись к фотографии;
- `alt` — точное описание изображения для доступности.

В массиве `ownerInterview` уже собраны подтверждённые владельцем сведения из интервью: 1995 год начала пути, результаты 2007 года и звание мастера спорта. Имя владельца и дополнительные детали можно добавить позже без изменения логики слайдера.

## Изображения

Исходные фотографии находятся в соседней папке `D:\Codex\F1\photo`. Оптимизированные WebP находятся в `public/images`, а сгенерированная единая фотосерия для первого экрана и карточек услуг — в `public/images/generated`. Чтобы заменить фотографию, добавьте оптимизированный файл в соответствующую папку и измените путь в `src/data.js` или `src/main.js`.

Англоязычные промты и параметры генерации сохранены в `image-generation-prompts.md`.

## Что ещё нужно получить от владельца

- реальные фотографии помещения автосервиса, ремонтных зон и выполненных работ: предоставленный набор содержит автоспортивные кадры, команду и награды;
- официальный домен — после получения нужно добавить абсолютный canonical URL, `og:url` и абсолютный URL для `og:image`;
- при необходимости — тексты юридической информации и политики обработки данных, если в будущем будет подключена отдельная серверная форма.

## Публикация

Содержимое `dist` можно разместить на любом статическом хостинге. Перед публикацией укажите реальный домен в SEO-метаданных, проверьте контактные данные владельца и настройте HTTPS.

Исходный `code.html`, `DESIGN.md` и `screen.png` оставлены без изменений как референсы экспорта Stitch и не входят в production-сборку.
